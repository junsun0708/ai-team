# Code Development

소프트웨어 개발 작업을 체계적으로 수행한다.

## 워크플로우

1. **요구사항 파악** — 요청에서 기능 요구사항을 추출한다.
2. **기존 코드 분석** — 대상 프로젝트의 구조와 기술 스택을 파악한다.
3. **구현** — 동작하는 최소 코드로 구현한다.
4. **검증** — 기본 동작을 확인한다.
5. **커밋** — 변경 사항을 커밋한다 (요청 시).

## 구현 원칙

- 동작하는 최소 단위로 구현한다. 불필요한 추상화를 만들지 않는다.
- 기존 코드 스타일을 따른다.
- 보안 취약점(하드코딩 시크릿, 인젝션 등)을 도입하지 않는다.
- 외부 의존성은 최소화한다.
- `.env`에 시크릿을 넣고, `.env.example`에 키 이름만 기록한다.

## 기술 스택별 가이드

### Python
- `requirements.txt`에 의존성 기록
- 가상환경 사용 권장 (`.venv/`)
- PEP 8 스타일 준수

### Node.js
- `package.json`에 의존성 기록
- ES modules 또는 프로젝트 기존 스타일 따름

### Shell Script
- `#!/bin/bash` shebang 포함
- `set -e`로 에러 시 즉시 중단
- 변수는 큰따옴표로 감싸기

## 커밋 규칙
- 요청 시에만 커밋한다.
- 커밋 메시지는 변경 내용을 간결하게 기술한다.
- `.env`, 시크릿 파일은 절대 커밋하지 않는다.
