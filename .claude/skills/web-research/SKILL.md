# Web Research

웹 검색을 통해 정보를 수집하고 구조화된 레포트를 작성한다.

## 워크플로우

1. **키워드 설계** — 사용자 요청에서 핵심 검색어를 추출한다. 한국어/영어 양쪽으로 검색한다.
2. **다각도 검색** — 최소 3개 이상의 소스에서 교차 검증한다.
3. **데이터 구조화** — 수집된 정보를 카테고리별로 분류한다.
4. **레포트 작성** — 한국어로 구조화된 레포트를 작성한다.

## 검색 전략

| 검색 유형 | 접근법 |
|----------|--------|
| 트렌드/현황 | 최신 기사, 보고서, 통계 사이트 우선 |
| 아이디어 발굴 | 커뮤니티(Reddit, HN, 디시인사이드 등), 포럼, 댓글 |
| 기술 비교 | 공식 문서, 벤치마크, 기술 블로그 |
| 시장/비즈니스 | 산업 보고서, 뉴스, 채용 공고 분석 |

## 레포트 형식

```markdown
# {조사 주제}

## 요약
{3줄 이내 핵심 요약}

## 상세 분석
### {카테고리 1}
- {발견 사항} (출처: {URL})
...

## 데이터 테이블
| 항목 | 값 | 출처 | 날짜 |
|-----|---|------|------|
...

## 출처 목록
1. {URL} - {사이트명} ({접근 일자})
...
```

## 품질 기준
- 모든 수치 데이터에 출처와 날짜를 명시한다.
- 상충되는 정보는 양쪽을 모두 기록하고 출처를 병기한다.
- 주관적 판단과 객관적 사실을 구분한다.
- 건수 기반 조사는 요청된 건수를 채울 때까지 반복 검색한다.
